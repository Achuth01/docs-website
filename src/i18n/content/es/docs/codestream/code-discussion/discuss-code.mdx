Lograr que los desarrolladores hablen sobre código antes y con más frecuencia durante el proceso de desarrollo genera excelentes resultados para su equipo. La conversación genera código de mejor rendimiento y mayor calidad, y desarrolladores más felices. New Relic CodeStream hace que sea más fácil que nunca hablar sobre código.

Ya sea una pregunta, una sugerencia, documentación o un informe de error, seleccione algún código, haga clic en <DoNotTranslate>**Add comment**</DoNotTranslate> o <DoNotTranslate>**Create issue**</DoNotTranslate> y, opcionalmente, compártalo en Slack o Teams. Todas estas discusiones se guardan y anclan a los bloques de código a los que hacen referencia, para que puedan aprovecharse en el futuro. Podría ser un nuevo desarrollador que se une al equipo, un desarrollador que intenta corregir un error en el código de otra persona o incluso simplemente usted tratando de recordar por qué realizó ese cambio hace seis meses. Cualquiera sea el caso, CodeStream le ayuda a comprender el código al mostrar las discusiones de forma contextual.

Incluso cuando un archivo cambia con el tiempo, los comentarios permanecen conectados al código. Agregue algunas líneas nuevas de código encima del bloque de código, edite el código o incluso corte y pegue el bloque completo en una sección diferente del archivo y verá que el comentario se mueve junto con los cambios.

## Crear un comentario o un problema [#create]

Seleccione un bloque de código en su editor y luego haga clic en los íconos <DoNotTranslate>**Add comment**</DoNotTranslate> o <DoNotTranslate>**Create issue**</DoNotTranslate> en el panel CodeStream junto a su selección.

Si está utilizando un IDE de JetBrains como IntelliJ, haga clic en el botón <DoNotTranslate>**+**</DoNotTranslate> que aparece en el margen del editor cuando selecciona un bloque de código.

Incluso cuando el panel CodeStream está cerrado o no está a la vista, puede discutir el código a través de las opciones de CodeStream en el menú contextual o de bombilla. También puede buscar el menú <DoNotTranslate>**+**</DoNotTranslate> en la parte superior del panel CodeStream.

## Compartir en Slack o Microsoft Teams [#sharing]

CodeStream tiene una [integración de Slack](/docs/codestream/code-discussion/slack-integration) y una [integración de Microsoft Teams](/docs/codestream/code-discussion/msteams-integration) que facilitan llegar a compañeros de equipo que no pasan mucho tiempo en el IDE o que aún no están en CodeStream. La integración de Slack incluso permite que tus compañeros de equipo respondan directamente desde Slack.

## Creando problemas [#issues]

Además de compartir con Slack y Teams, CodeStream también se integra con una serie de servicios de seguimiento de problemas. Si su equipo usa Asana, Azure DevOps, Bitbucket (nube), GitHub (nube o local), GitLab (nube o autoadministrado), Jira (nube o servidor), Linear, Shortcut, Trello o YouTrack (nube) Para problemas de seguimiento, puede crear un problema en uno de esos servicios directamente desde CodeStream. Simplemente seleccione el servicio que utiliza en el menú desplegable en la parte superior del formulario al crear un problema.

## Incorpore a la discusión a las personas adecuadas [#mentions]

Cuando creas un comentario o problema, CodeStream menciona automáticamente a las personas que tocaron más recientemente el código que estás comentando. Puede que sean las mejores personas para responder a tu pregunta, pero, por supuesto, puedes eliminar esas menciones y mencionar manualmente a otra persona si corresponde.

## Trabajar con diferentes versiones del código. [#code-versions]

Tal vez estés en una sucursal característica, tengas cambios locales o simplemente no hayas ingresado por un tiempo. Hay innumerables razones por las que el código que estás viendo puede ser diferente al que está viendo un compañero de equipo. Como resultado, habrá muchas ocasiones en las que el código al que se hace referencia en una discusión no coincida con el que tiene localmente.

CodeStream reconoce estas situaciones e incluye la versión original del bloque de código, la versión actual y una diferencia.

<img
  title="version diff"
  alt="A screenshot showing a code diff."
  src={codestreamCodemarkWithCodeDiff}
/>

Tenga en cuenta que con CodeStream puede analizar cualquier línea de código, en cualquier archivo fuente, en cualquier momento, incluso si se trata de un código que acaba de escribir en su editor y aún no ha confirmado. CodeStream le permite analizar el código en las primeras etapas del proceso de desarrollo.
