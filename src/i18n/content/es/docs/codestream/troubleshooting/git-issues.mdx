Cuando publica un mensaje con un bloque de código, puede encontrar una de las siguientes advertencias relacionadas con Git.

## El repositorio no está administrado por Git o no se puede encontrar Git [#not-found]

Si bien técnicamente no es un requisito, los elementos clave de la funcionalidad de Code Stream dependen de que su repositorio sea administrado por Git o un servicio de alojamiento de Git como GitHub. Si no estás usando Git, CodeStream no podrá conectar los comentarios y problemas que crees con los bloques de código apropiados en tus archivos fuente.

Este error también podría significar que Git no está en su RUTA. Si es así, agréguelo a su RUTA y luego reinicie su IDE.

## El repositorio no tiene una URL remota [#no-remote-url]

La URL remota es parte de lo que permite a CodeStream vincular el bloque de código de su mensaje al archivo y ubicación apropiados en el repositorio correspondiente en su IDE. Si CodeStream no puede identificar el repositorio que su compañero de equipo tiene abierto como el mismo en el que creó su comentario o problema, no podrá ver el comentario o problema.

Desde la línea de comando, use `git remote -v` para confirmar si tiene o no una URL remota configurada para su repositorio. Si no hay una URL remota, busque la URL correcta y luego especifique la URL remota a través de `git remote add origin <remote URL>`.

## Archivo no guardado [#unsaved]

Al igual que con los archivos no administrados por Git, CodeStream no podrá vincular un comentario o problema a un bloque de código en un archivo nuevo no guardado. Continúe y guarde el archivo antes de crear su primer comentario o problema.
