Al crear problemas de GitLab desde CodeStream, es posible que obtenga un error `404 Not Found` . Las URL de grupo/proyecto de GitLab se generan con una barra diagonal codificada que no es manejada por el proxy inverso apache2 predeterminado. Para permitir que Apache2 resuelva exitosamente estas URL, agregue lo siguiente a su configuración de Apache2:

* Agregue AllowEncodedSlashes
* Agregue `nocanon` al final de su línea ProxyPass

Por ejemplo:

```
AllowEncodedSlashes NoDecode
ProxyPass / http://33.211.55.3:9000/ nocanon
```
